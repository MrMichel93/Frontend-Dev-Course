## Exercise 1: Build a Profile Card

Create a profile card component with:
- Image at the top (rounded)
- Name and title
- Brief description
- Social media buttons
- Shadow and hover effects

**Requirements:**
- Use Flexbox or Grid
- Apply appropriate spacing
- Add hover effects
- Make it responsive

## Exercise 2: Create a Navigation Bar

Build a responsive navigation bar with:
- Logo on the left
- Menu items on the right
- Hamburger menu for mobile (you can simulate this visually)
- Hover effects on links
- Sticky positioning

## Exercise 3: Design a Pricing Table

Create a pricing section with three tiers:
- Free, Pro, and Enterprise plans
- Different styling for featured plan
- List of features for each
- Call-to-action buttons
- Responsive grid layout

## Exercise 4: Build a Contact Form

Create a contact form with:
- Name input field
- Email input field
- Message textarea
- Submit button
- Proper focus states
- Form validation styling (error/success states)

## Exercise 5: Create a Blog Card Grid

Build a grid of blog post cards:
- 1 column on mobile
- 2 columns on tablet
- 3 columns on desktop
- Each card has image, title, excerpt, and read more link
- Hover effects on cards

## Exercise 6: Design an Alert Component

Create alert components for:
- Success messages (green)
- Error messages (red)
- Warning messages (yellow)
- Info messages (blue)
Each with an icon and close button

## Exercise 7: Build a Dashboard Sidebar

Create a sidebar layout:
- Fixed sidebar on desktop
- Collapsible on mobile
- Navigation items with icons
- Active state styling
- Main content area

## Bonus Challenge: Complete Landing Page

Build a complete landing page with:
- Hero section with CTA
- Features section (3-column grid)
- Testimonials carousel
- Pricing table
- Footer with links
- Fully responsive
- Smooth animations and transitions

## Tips

- Start mobile-first
- Use consistent spacing
- Test on different screen sizes
- Reference the Tailwind documentation
- Experiment with different utilities

## Solutions

Try to complete exercises before looking at solutions. You can find reference implementations in the `examples/` directory of this module.
